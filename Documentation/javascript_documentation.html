Introduction to JavaScript

JavaScript is a high-level computer language used for programming or scripting. It allows the implementation of features on web pages. It is a cross-platform, object-oriented language that can be connected to objects in its environment and dynamically manipulate web pages.

MDN describes Javascript in the following way:

"JavaScript is a scripting or programming language that allows you to implement complex features on web pages — every time a web page does more than just sit there and display static information for you to look at — displaying timely content updates, interactive maps, animated 2D/3D graphics, scrolling video jukeboxes, etc. — you can bet that JavaScript is probably involved. It is the third layer of the layer cake of standard web technologies, two of which (HTML and CSS) we have covered in much more detail in other parts of the Learning Area."

What is JavaScript used for?

JavaScript can be used on the client-side or the server-side and allows a site to be interactive. HTML and CSS generally provide the structure and style of a page while JavaScript provides functionality.

  • Client-side JavaScript involves supplying objects to control a browser and its Document Object Model (DOM). Client-side interactions include: inserting HTML into the page and responding to user events such as mouse clicks, form input and page navigation.
  • Server-side JavaScript involves supplying objects relevant to running JavaScript on a server. Server-side interactions all an application to communicate with a database, provide continuity of informaiton from one invocatioon to another of the application, or perform file manipulations on a server.

Variables

Variables are used to store values. To create and use a variable, you must first declare and initialize your variable. Your varible will also need a name, known as an identifier.

Declaring Variables

To declare a variable, you must use one of the following keywords:

  • var
  • let
  • const

The var keyword is used for declaring variables that are available to both the local and the global scope. You should use caution when using this keyword in cases where you don't want to accidentally overwrite the variable.

The let keyword, introduced in ES6, can also store values, but is only available to the Block Scope within which it was declared. Additionally, a let variable cannot be redeclared, but can be reassigned; it will throw an error instead.

The const variable, also introduced in ES6, can store values, but has a stricter ruleset. The const is only available in the Blockk Scope in which it was declared and cannot be redeclared, just like the let keyword. But the const keyword cannot be reassigned. According to W3Schools the const variable should be your first choice unless you know that the value will change.

You should always use the const variable when you declare:

  • A new Array
  • A new Object
  • A new Function
  • A new RegExp

Initializing Variables

To initialize a variable, you have to use one of the variable keywords, var, let or const followed by an identifier. Also, you can use two forward slashes // to add comments.

For example:

var name = Tom; // For variables outside of the Block Scope

for (let i = 0; i < 3; i++) {
 console.log("Hello"); // Prints 'Hello' 3 times to the console.
}

const PI = 3.14;

You can also declare a variable first and initialize it later.

Example:

var name;
name = "John Doe";

Rules for naming a variable

  1. A name should start with a letter, an underscore or a dollar sign.
  2. A name cannot start with a number
  3. A name is case-sensitive
  4. A name can only contain letters, digits, underscores and dollar signs.
  5. JavaScript reserved keyords (var, function, if, else, etc) cannot be used as variable names.

Data Types

Data types can either be primitive (meaning that they can only take on one value at a time) or complex (meaning that they can consist of multiple values grouped together).

Primitive data types:

  • String
  • Number
  • Boolean
  • Undefined
  • Null

Complex data types

  • Objects
  • Functions

String data type

Uses characters to represent text. Can use double quotes "" or single quotes ''.

var name = "John";
var lastName = 'Doe';

Number data type

This data type represents an integer or a floating point.

var num = 23;
var floatingNum = 43.55;

Numbers and Strings can be combined with the + operator. But be careful. If the string comes before the number when they are concatenated, the number will be turned into a string.

var num1 = 11;
var num2 = 22;
var name = Tom;

console.log(num1 + num2 + ": " + name); // 33: Tom
console.log(name + ": " + num1 + num2); // Tom: 1122

Boolean data type

Returns one of two values. true or false.

var happy = true;
var rich = false;

You can use comparison operators to test values against each other and returna boolean.

var areTheyEqual = (3 == 4);
var isItBigger = (3 > 4);
var isItSmaller = (3 < 4);

console.log(areTheyEqual); // false
console.log(isItBigger); // false
console.log(isItSmaller); // true

*There will be more information on comparison operators in the section that discusses operators.

Undefined data type

Represents a value that is not assigned.

var bib;
console.log(bib); // undefined

Null data type

A special data type representing nothing. You can empty a variable by setting its value to null.

var x = 10;
x = null;

console.log(x); // null

Object data type

This is a collection of related data. These are the following rules for the object data type:

var myObject =
{
  firstName : 'John',
  lastName : 'Doe',
  age : 23
};

The data type of an array is object.

*There will be more information on objects in the section that discusses objects.

Function data type

MDN states:

A function in JavaScript is similar to a procedure--a set of statements that performs a task or calculates a value, but for the procedure to qualify as a function, it should take some input and return an output where there is some obvious relationship between the input and the output.

function square(number) {
  return number * number; }

*There will be more information on functions in the section that discusses functions.

Data types extra notes

The typeof operator returns a string indicating the data type of the evaluated operand. Objects, arrays and null will all return object. A function will return function.
JavaScript is dynamic. Meaning that one variable can represent different data types if necessary.

var num = 33;
console.log(num); // 33

num = "I'm a string now!";
console.log(num); // I'm a string now!

Operators

There are four main types of operators in JavaScript. But ES6 introduces the conditional or ternary operator. Operators can be used to return values or manipulate variables.

  1. Arithmetic operators
  2. Assignment operators
  3. Comparison operators
  4. Logical operators

Arithmetic operators

Arithmetic operators: +, -, *, /, %

var num1 = 10 + 20; // 30
var num2 = 10 - 5; // 5
var num3 = 10 * 2.5; // 25
var num4 = 10 / 3; // 3. Because this operator doesn't show the remainder.
var num5 = 10 % 3; // 1. The modulo operator only returns the remainder.

Assignment operators

Assignment operators:
= Is used to assign a variable to a value.
+= Is used to add a value to an existing variable.
-= Is used to subtract a value from an existing variable.
*= Is used to multiply an existing variable by a value.
/= Is used to divide an existing variable by a value.
%= Is used to assign the remainder to a variable.
**= Assigns the exponent of a value to the variable.

var num1 = 3; // Assigns 3 to num1.
var num1 += 2; // 5
var num1 -= 3; // 2
var num1 *= 10; // 20
var num1 /= 2; // 10
var num1 %= 4; // 2
var num1 **= 3; // 8

NOTE: When useing the + or += operators with strings and numbers, you will sometimes have a string returned. Just as in the example shown in the string variable explanation above.
There are also some more advanced assignment operators that will not be addressed in this section.

Comparison operators

Comparison operators:
== Equality operator.
!= Inequality operator.
=== Strict Equality operator.
!== Strict Inequality operator.
< Greater than.
<= Greater than or equal to.
> Less than.
>= Less than or equal to.

console.log(1 == 1); // Returns true.
console.log(1 != 1); // false
console.log(1 === '1'); // false. Because 1 is a number and '1' is a string.
console.log(1 !== '1'); // true
console.log(1 < 2); // true
console.log(1 <= 2); // true
console.log(1 > 0); // true
console.log(1 >= 1); // true

NOTE: When useing the + or += operators with strings and numbers, you will sometimes have a string returned. Just as in the example shown in the string variable explanation above.
There are also some more advanced assignment operators that will not be addressed in this section.

Incrementing and decrementing variables

Variables can be incremented or decriemented in JavaScript by using ++ or -- respectively.

If/Else Statements

If/Else Statements are used to test if certain conditions are true. They go in sequential order, so be sure to order the conditions properly.

// BEWARE! The following example will not work properly because of the order of the if/else statements.

var x = 18;
if (x > 13) {
 console.log("You are a teenager."); // This will be the output of this example.
} else if (x >= 18) {
 console.log("You are an adult."); // This line can never be reached.
} else {
 console.log("You are not a teenager or an adult.");
}

In the above example, the (x >= 18) condition will never be reached because anything that is greater than 18 is also greater than 13. And since (x > 13) is placed above the 18+ condition, it will execute first and not continue to the next condition.

Another issue with the above code is that the variable is not very descriptive. x is not a very meaningful variable. So, we will fix that in the next example.

var x = 18;
if (x >= 65) {
 console.log("You can definitely drink in the US.");
} else if (x >= 21) {
 console.log("You are an adult and you can drink in the US.");
} else if (x >= 18) {
 console.log("You are an adult."); // This line will be the output of this code.
} else {
 console.log("You are not an adult. Don't even think about drinking.");

There, that's better. Now the conditions are in such an order that they do not overlap. And, in this case, we do not have to specify a condition for anyone less than 18. The else statement covers that. We could add such a condition and use the else statment to catch errors or input that does not match a number. But this will be covered in a later section.

Objects

An object is made of properties and their values. There are many ways to make an object, and actually, basically everything in JavaScript is an object anyway.

In the following example, you will see a variety of key: value pairs.

Ex.

var myObj = {
 make: "Ford",
 model: "Mustang",
 year: 1969,
 owner: {
  firstName: "John",
  lastName: "Doe",
  age: 23,
  hobbies: ["games", "food", "movies"]
 },
 carDetails: function () {
  for(let i = 0; i < 3; i++) {
   console.log(i+1);
  }
 }
};

As you can see in the example above, you can store just about anything you want in an object. From simple strings to numbers and even arrays or functions. You can even store objects within objects!

If you want to access the values within an object, you can use dot notation:

console.log(myObj.make); // Ford
console.log(myObj.owner.age); // 23

// Notice that you can chain together dot notation to access an object within an object.

Or, you can access the values using bracket notation:

console.log(myObj["make"]); // Ford
console.log(myObj["owner"]["age"]); // 23

// Make sure to include single quotes '' or double quotes "" when using bracket notation.

That was super fun! Now let's try another little trick we can use with objects.

If you want to just access all of the keys of an object, you can use the Object.keys() method.

console.log(Object.keys(myObj)); // ["make", "model", "year", "owner", "carDetails"]

And the same goes for the values of an object. You can use the Object.values() method.

console.log(Object.values(myObj)); // ["Ford", "Mustang", 1969, {…}, ƒ]

In the above example, {…} represents an object within our object. And ƒ represents a function within our object.

The syntax for objects in JavaScript is the same as object literals in JSON/AJAX. JSON/AJAX has already been implemented in rendering part of this webpage, but we won't discuss this until later.

Functions

A function in JavaScript is similar to a procedure--a set of statements that performs a task or calculates a value, but for a procedure to qualify as a function, it should take some input and return an output where there is some obvious relationship between the input and the output.

Function Declaration

There are three parts to defining a function.

  1. The name of the function.
  2. The parameters of the function.
  3. The statements that define the function, enclosed in curly brackets.

Here is an example of a simple function name hello that takes one parameter, name, and logs this greeting to the console.

However, it should be noted that functions do not really require any of these things to be considered a function. More on this in the coming sections.

function hello(name) {
 console.log("Hello " + name + "!");
}

Notice that we use the function keyword when declaring a function.

Function Calling

So, we have everything set up. We have a name, we have parameters and we have a statment within our funciton. So, what now? Now we call the function. Calling a function, or executing a function, is how we actually use it.

To call a fuction, we are going to need to make a reference to it. That's as simple as using the name of the function, or the variable storing the function, followed by parentheses ().

function hello() {
 console.log("Hello!");
}
function greeting(name) {
 console.log("Hello, " + name + "!")
}

hello(); // Hello!
greeting("Theodore"); // Hello, Theodore!

Notice that we can have functions with or without parameters, but if you do not enter a name into the second example, you will get a response of Hello, undefined!

Function Expressions

As we said, functions do not have to have a name. They can be what is known as an anonymous function.

Ex.

const square = function(num) {
 return num * num;
}
var x = square(2); // 4

In the following example, we can see how an anonymous function can be referenced by another function.

const cube = function (num) {
 return num * num * num;
}

function getCubeFunc(func, num) {
 console.log("The cube of " + num + " is " + cube(num) + ".") ;
}

getCubeFunc(cube, 4); // The cube of 4 is 64.

These functions without names can be useful at times, but sometimes you are going to want to reference the function within itself. For that, we are going to need to give it a name.

const sumFunc = function sumAll(num) {
 if (Number.isInteger(num) && num > 0) {
  return num + sumAll(num - 1) || 1;
 }
}

console.log(sumFunc(5)); // 15

This example, which is not an anonymous function, makes a reference to itself so that it will run multiple times until the if condition is no longer true. It is meant to take an integer and add all integers that are smaller than it.

*The || statement is used in the previous example because the last value is not added properly without it. This can also be fixed by using an else statement, but is not quite as concise.

The Number.isInteger(num) method makes sure that the number given is an integer. Also note that the function will only work if a valid number greater than 0 is entered.

Arrays

In JavaScript, an array is an ordered list of values. Each value is called an element specified by an index. An array can hold values of different types, because it's flexible that way. You can store a string, a number and a boolean all in the same array if you want to. You also don't need to specify the size of the array right away because the size is dynamic and depends on the values that will be used.

Recursion

Recursion is a process that calls itself. A function that calls itself is called a recursive function.

A recursive function must have a condition to stop calling itself. Otherwise, the function is called indefinitely. Once this condition is met, the function will stop calling itself. This is called a base condition.

Event Listeners

An event listener is a procedure in JavaScript that waits for an event to occur. This can be a mouse click/press or released key/ or any number of events.

Modifying HTML and CSS

Normally, HTML and CSS are enough for the structure and style of a page, but sometimes you will want to do something that requires tools that are not present in these languages. That's where JavaScript comes in.

For example, when building a collapsable dropdown menu with HTML and CSS, it makes sense that the menu should collapse if you click anywhere on the screen that makes the dropdown lose focus, or blur (more on this later).

HTTP Protocols

Hypertext Transfer Protocol, or HTTP for short, is an application-layer protocol for transmitting hypermedia documents, such as HTML. It was designed for communication between web browsers and web servers, but it can also be used for other purposes.

AJAX

AJAX (Asynchronous JavaScript And XML) is the use of the XMLHttpRequest object to communicate with servers. It can send and receive information in various formats, including JSON, XML, HTML, and text files. AJAX is not a programming language.
